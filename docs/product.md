# DocGraph 제품 명세

Notion 문서 간 의존 관계를 그래프로 모델링하고, 변경 전파와 정합성 위반을 자동 감지하는 SaaS.

## 사용자 역할

| 역할 | 범위 | 권한 |
| --- | --- | --- |
| 워크스페이스 소유자 | 워크스페이스 | 멤버 초대, 프로젝트 생성. 자동으로 모든 프로젝트의 Project Admin 권한을 가진다. |
| Project Admin | 프로젝트 | 문서 타입·룰 편집, 담당자 설정, Webhook 설정, 프로젝트 멤버 역할 관리. Project Admin이 다른 멤버를 Project Admin으로 지정할 수 있다. |
| 멤버 | 프로젝트 | 그래프 조회, 정합성 위반 확인, 충돌 해소, 인박스 조회. 배정된 프로젝트에만 접근할 수 있다. |

## 구현 우선순위

최종 발표 데모 기준으로 기능을 세 단계로 분류한다. 데모 핵심 플로우는 **그래프 화면 중심**으로 진행한다: `Notion 변경 → 영향 엣지 식별 → 정합성 검증 → 수정 제안 → Notion 반영`.

### P0 — Demo Critical

데모에서 DocGraph의 핵심 가치를 증명하기 위해 반드시 구현한다.

- Notion OAuth 및 워크스페이스 연결
- 프로젝트 생성 및 초기 동기화
- 문서 타입 매핑 (기본 타입)
- 링크·멘션 기반 의존 관계 그래프 생성
- 문서 변경 Webhook 감지
- 영향 엣지 식별
- AI 정합성 검증
- 충돌 엣지 빨간색 표시
- 문서 비교 패널
- AI 수정 제안
- AI 제안 승인 후 Notion 반영

### P1 — Demo Quality

핵심 플로우를 더 자연스럽게 만든다. P0 완료 후 진행한다.

- 인박스 (내 담당 충돌 목록)
- 타입별 기본 담당자
- 무시 처리 (충돌 수동 마킹)
- 키워드 매칭 기반 연결 제안·점선 엣지 (데모 그래프가 빈약한 경우에만)
- 프로젝트 멤버 배정 및 역할 관리

### P2 — Post-MVP

제품 완성도에 필요하지만 최종 발표 데모 범위에서 제외한다.

- Slack/Discord Webhook 알림
- 사용자 비교 뷰 직접 편집 후 Notion 반영
- 커스텀 룰·엣지 편집 UI
- 대시보드 통계
- 개별 문서 담당자 오버라이드
- Notion 담당자 속성 자동 로드
- 연결 제안: 임베딩 기반 의미 유사도 매칭으로 고도화

## 기능

### F1. Notion 연동

- Notion OAuth를 통해 워크스페이스를 연결할 수 있어야 한다.
- 연결된 워크스페이스에서 루트 페이지를 프로젝트로 등록할 수 있어야 한다. 하나의 워크스페이스에 여러 프로젝트를 등록할 수 있다.
- 등록된 프로젝트 루트 페이지 하위 트리에 변경이 발생하면 Notion Webhook 이벤트를 수신해야 한다.
- 이벤트 수신 후 Notion API를 호출해 변경된 페이지의 전체 내용을 조회할 수 있어야 한다.

### F2. 문서 타입 분류

- 프로젝트 설정에서 상위 페이지와 문서 타입을 매핑한다. 동기화된 문서는 직계 상위 페이지의 매핑을 조회해 타입을 결정한다.
- 매핑되지 않은 상위 페이지의 문서는 미분류 상태로 저장되며, Admin이 수동으로 타입을 지정해야 한다.
- 기본 제공 타입은 다음과 같다.
  - `meeting_notes`
  - `planning`
  - `requirements`
  - `design`
  - `research`
- Project Admin은 프로젝트별로 타입을 추가·삭제·수정할 수 있어야 한다.

### F3. 의존 관계 그래프

- Notion 페이지 링크·멘션이 감지되면 해당 연결을 의존 관계로 자동 등록해야 한다. 링크 대상이 같은 프로젝트 외부 문서인 경우 의존 관계로 등록하지 않는다.
- 등록된 의존 관계의 문서 타입 쌍이 룰 테이블에 해당하면, 해당 룰의 체크 항목으로 정합성 검증을 수행한다.
- 기본 제공 룰은 다음과 같다.

| 출발 | 도착 | 체크 항목 |
| --- | --- | --- |
| `meeting_notes` | `planning` | 결정사항 반영 여부 |
| `meeting_notes` | `requirements` | 결정사항 반영 여부 |
| `research` | `planning` | 전제 조건 유지 여부 |
| `planning` | `requirements` | 범위 일치 여부 |
| `requirements` | `design` | 스펙 일치 여부 |

- 생성된 의존 관계를 시각적 그래프로 조회할 수 있어야 한다.
- (P1) Notion 링크가 없더라도 룰 타입 조합에 해당하는 문서 쌍 중 키워드 매칭 상위 항목을 연결 후보로 제안해야 한다. 제안은 문서 그래프에서 점선 엣지로 표시되며, Admin이 수락하면 실선 엣지로 전환된다.
- (P2) Admin은 의존 관계를 수동으로 추가·삭제할 수 있어야 한다.

### F4. 정합성 검증

- 문서에 변경이 감지되면 의존 관계로 연결된 문서와의 정합성을 AI로 검증해야 한다.
- AI 검증 결과로 타깃 문서의 충돌 구간, 원인, 수정 제안을 반환받아야 한다.
- 정합성 위반이 감지된 연결 엣지는 그래프에서 빨간색으로 표시되어야 한다.
- 충돌 구간은 인라인으로 표시되어야 하며, 수정 제안을 함께 제공해야 한다.
### F5. 알림

- 정합성 위반이 감지되면 문서 그래프에 미해소 위반 항목을 표시해야 한다. (P2) 대시보드에서도 집계 표시한다.
- 위반 항목은 해소 완료 전까지 지속적으로 표시되어야 한다.
- (P2) 정합성 위반이 감지되면 프로젝트에 설정된 Webhook URL로 알림을 발송할 수 있어야 한다.
- (P2) Webhook 알림은 Slack, Discord 등 일반적인 Webhook 수신 서비스와 호환되어야 한다.
- (P2) Webhook URL은 프로젝트별로 설정할 수 있어야 한다.
- (P2) Webhook 알림 발송이 실패하더라도 인앱 위반 표시는 유지되어야 한다.

### F6. 충돌 해소

- 충돌이 감지된 문서 쌍은 이분할 비교 뷰로 조회할 수 있어야 한다.
- AI 수정 제안을 승인하면 Notion integration 권한으로 해당 내용을 Notion 문서에 직접 반영한다. 승인 주체는 로그인 사용자로 기록된다.
- 반영 전 대상 문서의 최종 수정 시각을 비교해 제안 생성 이후 변경이 있으면 적용을 중단하고 재검증을 요구한다.
- Notion 쓰기 실패 시 권한 오류로 안내하고 충돌은 유지된다.
- 사용자는 AI 수정 제안을 거부할 수 있어야 하며, 거부 시 위반 항목은 유지된다.
- (P1) 사용자는 정합성 위반 항목을 `무시` 상태로 마킹할 수 있어야 한다. 무시된 항목은 해당 문서 쌍 중 하나가 외부에서 변경되면 자동으로 해제된다.
- (P1) 무시된 항목은 그래프에서 시각적으로 구분되어야 한다.
- (P2) 사용자는 비교 뷰에서 직접 내용을 수정해 Notion에 반영할 수 있어야 한다. AI 제안 승인과 동일하게 반영 전 최종 수정 시각을 확인해 변경이 있으면 중단한다.

### F7. 담당자 (P1)

- Project Admin은 프로젝트 내 문서 타입에 기본 담당자를 지정할 수 있어야 한다. 해당 타입으로 분류된 문서는 기본 담당자를 자동으로 상속한다.
- 충돌 수정 책임은 도착 문서(target)의 담당자에게 귀속된다. target 담당자가 없으면 해당 프로젝트의 모든 Project Admin에게 귀속된다.
- 출발 문서(source) 담당자는 알림 대상이 아니며 비교 뷰에서 참조 정보로만 표시한다.

### F8. 인박스 (P1)

- 내가 담당자인 문서가 target인 충돌 목록을 내가 배정된 모든 프로젝트에 걸쳐 조회할 수 있어야 한다.
- 각 항목에서 문서 비교 패널로 바로 이동할 수 있어야 한다.
- 상태별 필터를 제공해야 한다: 미해소 / 무시 / 전체

## 화면 구성

```text
랜딩
  ↓ (로그인)
Notion OAuth2
  ↓
인박스 (기본 화면, P1)               ← 전체 워크스페이스 대상
  └─ 문서 비교 패널 (항목 클릭 시)

워크스페이스 목록                     ← 내비게이션으로 이동
  ├─ 워크스페이스 설정
  └─ 프로젝트 목록
       ├─ 프로젝트 설정
       │    └─ 멤버 역할 관리
       ├─ 문서 그래프
       │    ├─ 문서 패널 (노드 클릭 시)
       │    ├─ 연결 제안 패널 (점선 엣지 클릭 시, P1)
       │    ├─ 엣지 패널 (일반 엣지 클릭 시)
       │    └─ 문서 비교 패널 (충돌 엣지 클릭 시)
       ├─ 타입 그래프 (P2)
       │    ├─ 타입 패널 (노드 클릭 시)
       │    └─ 룰 패널 (엣지 클릭 시)
       └─ 대시보드 (P2)
```

### 랜딩

- 비로그인 상태: 서비스 소개 콘텐츠 표시
- 로그인 상태: 프로젝트 목록으로 자동 리다이렉트. (P1) 인박스 구현 시 인박스로 이동.

### 로그인

- Notion 계정으로 OAuth2 인증
- 로그인 성공 시 프로젝트 목록으로 이동. (P1) 인박스 구현 시 인박스로 이동.
- 별도 회원가입 플로우 없음. 첫 OAuth 완료 시 Notion 사용자 정보로 DocGraph 계정을 자동 생성한다.

### 워크스페이스 목록

Notion OAuth로 연결된 계정이 보유한 워크스페이스 목록.

- **내 워크스페이스**: 내가 소유한 워크스페이스 목록
- **참여 중인 워크스페이스**: 멤버로 초대받은 워크스페이스 목록
- 각 워크스페이스 클릭 시 프로젝트 목록으로 이동
- 워크스페이스 추가: Notion OAuth를 추가 실행해 새 워크스페이스를 연결한다.

멤버 초대 및 관리는 워크스페이스 설정에서 처리한다.

### 워크스페이스 설정

워크스페이스 단위 설정을 관리하는 페이지. 워크스페이스 생성자 전용.

| 항목 | 설명 |
| --- | --- |
| 멤버 관리 | 멤버 초대(Notion 멤버 선택 또는 이메일), 목록 조회, 제거. 초대 방법에 관계없이 모두 동등한 멤버로 참여한다. |

### 프로젝트 목록

워크스페이스 내 내가 접근 가능한 프로젝트 목록. P0에서는 내가 생성한 프로젝트를 표시하고, (P1) 배정된 프로젝트만 표시한다.

- 각 프로젝트 클릭 시 문서 그래프로 이동
- 프로젝트 추가: Notion 워크스페이스 내 루트 페이지를 선택해 등록 (워크스페이스 생성자 전용)

### 프로젝트 설정

프로젝트 단위 설정을 관리하는 페이지. Project Admin 전용.

| 항목 | 설명 |
| --- | --- |
| 정합성 관리 활성화 | 해당 프로젝트의 문서 변경 감지 및 정합성 검증 활성화 여부 |
| Webhook URL (P2) | 충돌 감지 시 알림을 받을 Slack·Discord 등 Webhook URL |
| Webhook 연결 상태 (P2) | 설정된 Webhook의 정상 연결 여부 표시 |
| 멤버 배정 (P1) | 워크스페이스 멤버를 프로젝트에 추가·제거. 배정된 멤버만 해당 프로젝트에 접근할 수 있다. |
| 멤버 역할 관리 (P1) | 프로젝트 내 멤버별 역할(Project Admin / 멤버) 지정 |

### 인박스 (P1)

내가 배정된 프로젝트에 걸쳐 내가 담당자인 문서가 target인 충돌 목록.

- 각 항목: 워크스페이스명·프로젝트명, 충돌 문서 쌍, 충돌 요약, 발생 시각
- 항목 클릭 시 해당 프로젝트의 문서 비교 패널로 이동
- 상태 필터: 미해소 / 무시 / 전체

### 문서 그래프

프로젝트 내 문서 의존 관계를 시각화하는 메인 화면.

- 문서를 노드, 의존 관계를 엣지로 표시
- 정합성 위반이 감지된 엣지는 빨간색으로 표시

**문서 패널** (노드 클릭 시)

- 문서 타입 표시 및 수동 수정
- (P1) 담당자 표시 (타입 기본값 상속 여부 표시)
- 해당 문서의 의존 관계 목록 표시 (수동 추가·삭제 가능)

**연결 제안 패널** (점선 엣지 클릭 시, P1)

- 연결된 두 문서를 이분할로 표시
- 수락: 실선 엣지로 전환, 정합성 검증 트리거
- 무시: 제안 삭제

**엣지 패널** (일반 엣지 클릭 시)

- 출발·도착 문서명 및 타입
- 적용 중인 룰과 체크 항목
- 마지막 검증 시각 및 결과

**문서 비교 패널** (충돌 엣지 클릭 시)

- 연결된 두 문서를 좌우 이분할로 표시 (git diff 스타일)
- 충돌 구간 인라인 하이라이트
- AI 수정 제안 표시
  - 승인: Notion 문서에 직접 반영
  - 거부: 제안 닫기, 위반 항목은 유지
  - (P2) 사용자가 패널 내에서 직접 수정 후 Notion에 반영
- (P1) 위반 무시 처리: 수동으로 `무시` 마킹 가능

### 타입 그래프 (P2)

프로젝트 내 문서 타입 간 정합성 룰을 시각화하는 화면. 타입을 노드, 룰을 엣지로 표시한다.

- 모든 역할이 열람 가능

**타입 패널** (노드 클릭 시)

- 타입명, 설명
- 기본 담당자
- Project Admin이면 편집 가능

**룰 패널** (엣지 클릭 시)

- 출발·도착 타입
- 체크 항목
- Project Admin이면 편집 가능

**Project Admin 전용**

- 문서 타입 추가·삭제·수정
- 타입별 기본 담당자 지정
- 룰 추가: 타입 노드 간 엣지를 그어 추가, 체크 항목 입력
- 룰 삭제: 엣지 선택 후 삭제
- 체크 항목 수정: 엣지 클릭 후 편집

### 대시보드 (P2)

프로젝트별 충돌 현황을 집계·기록하는 별도 페이지.

- 위반 감지 건수 통계
- 미해소 위반 건수
- 위반 발생·해소 이력
- 기간 필터는 최근 30일 고정 (추후 확장 가능)

## 범위 제외

- 범용 인앱 문서 에디터 (충돌 해소 목적의 제한적 인라인 편집은 포함)
- 프로젝트 간 정합성 검증
- GitHub 연동
- Notion 외 문서 소스 연동 (Confluence 등)
- Webhook 외 별도 동기화 방식 (주기적 폴링 등)
